## 4. 🔒 Security & Cryptography

### 🔹 Cryptographic Primitives
HAFA employs industry-standard cryptographic primitives:
- **Ed25519 Signatures:** Quantum-resistant digital signatures
- **SHA3-256 Hashing:** Secure hash function for block headers and Merkle trees
- **ChaCha20-Poly1305:** Authenticated encryption for private key storage
- **Zeroize:** Automatic secure memory cleanup to prevent key leakage

### 🔹 Multi-Signature Support
The protocol supports multi-signature wallets for enhanced security and DAO governance:
- Configurable threshold (e.g., 2-of-3, 3-of-5)
- Required for high-risk evolution proposals
- Enables decentralized decision-making

### 🔹 Checksum Addresses
Ethereum-style checksum addresses prevent typos and address manipulation:
- Format: `pubkey:checksum`
- Checksum: First 4 bytes of SHA3-256 hash
- Validation: Automatically verified by `Address::from_hex()`

### 🔹 Encrypted Key Storage
Private keys can be encrypted with user-provided passphrases:
- Encryption: ChaCha20-Poly1305 with random nonce
- Key derivation: SHA3-256(passphrase + salt)
- Storage: Encrypted keys can be safely stored on disk

### 🔹 Secure Memory Handling
The `KeyPair` struct implements `Zeroize` and `ZeroizeOnDrop`:
- Automatic cleanup when key goes out of scope
- Prevents key material from lingering in RAM
- Protects against memory dumps and cold boot attacks

    "cpu_percent": 45.0,
